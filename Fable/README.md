# Fable — Adaptive Traffic-Signal Control with Reinforcement Learning

A parameter-shared Double DQN agent that controls 16 signalised intersections in a SUMO-simulated 4×4 grid network. Built for CPU-only nodes with a focus on sample efficiency and reproducible evaluation.

## Setup

### Prerequisites

- Python 3.9+
- A GNU/Linux environment (the SUMO binary distribution is OS-specific)
- ~3 GB free disk space

### Install

```bash
# Clone the repository
git clone <repo-url> /path/to/Fable
cd /path/to/Fable

# Create a virtual environment
python3 -m venv venv
source venv/bin/activate

# Install all dependencies
pip install eclipse-sumo==1.27.1 libsumo==1.27.1 sumolib==1.27.1
pip install torch==2.13.0+cpu --extra-index-url https://download.pytorch.org/whl/cpu
pip install numpy==2.5.1 pandas==3.0.3 matplotlib==3.11.0 psutil==7.2.2 PyYAML==6.0.3 pytest==9.1.1 pillow==12.3.0

# Or install from requirements.txt
# pip install -r requirements.txt
```

### Verify installation

```bash
source venv/bin/activate
python -c "import libsumo, torch, numpy, pandas, matplotlib, psutil, yaml; print('All dependencies OK')"
```

## Usage

### 1. Train the agent

```bash
source venv/bin/activate
cd /path/to/Fable

# Train from scratch (3063 episodes, ~2 hours)
python train.py

# Resume training from the latest checkpoint
python train.py --resume

# Train for a specific number of episodes
python train.py --episodes 100

# Use a custom config
python train.py --config my_config.yaml
```

Training output:
- **Logs**: `logs/train.jsonl` (structured JSONL) and `logs/train_full.out` (human-readable stdout)
- **Checkpoints**: `checkpoints/model_best.pt` (best reward) and `checkpoints/model_latest.pt` (resumable latest)

### 2. Evaluate the trained model

```bash
source venv/bin/activate
cd /path/to/Fable
python evaluate.py
```

This runs the trained checkpoint (`checkpoints/model_best.pt`) on 8 scenarios × 20 deterministic seeds = 160 episodes using 2 parallel worker processes. Results:

- **`results/metrics.csv`**: 160 rows (one per episode), 11 metric columns, status flags
- **`results/aggregate.json`**: Per-scenario and overall aggregates with means and standard deviations
- **`results/spatial.json`**: Per-edge and per-intersection averages for visualisation
- **`logs/eval.jsonl`**: Structured episode log

To evaluate a specific checkpoint or scenario:

```bash
python evaluate.py --checkpoint checkpoints/model_best.pt --scenarios normal high_demand
```

### 3. Visualise results

```bash
source venv/bin/activate
cd /path/to/Fable
python visualize.py
```

Produces `results/final_visualization.png` (1600×1600 px, deterministic):

- Road colour = average congestion (0–100% occupancy)
- Road width = average queue length (0–40 vehicles)
- Intersection marker size = average waiting time (0–180 s)
- Arrows = dominant traffic flow direction
- Red glow zones = bottleneck/gridlock areas (congestion > 60%)
- Text overlay = algorithm name, avg waiting time, avg queue length, throughput, gridlock %

### 4. Run tests

```bash
source venv/bin/activate
cd /path/to/Fable
python -m pytest tests/ -v
```

Expected: 16 tests passing covering metric aggregation, invalid-episode handling, image dimensions, clipping behaviour, and deterministic rendering.

### 5. Benchmark simulation speed

```bash
source venv/bin/activate
cd /path/to/Fable
python scripts/benchmark.py
```

Measures simulation steps/second on the current node. Results stored in `results/benchmark.json`.

## Configuration

All tunable parameters are in `config.yaml`:

| Section | Key Parameters |
|---|---|
| `env` | Decision interval (5 s), yellow duration (3 s), min green (5 s), observation dimension |
| `reward` | Pressure coefficient (0.05), waiting coefficient (0.02) |
| `agent` | Hidden sizes [128,128], discount 0.95, learning rate 5e-4, buffer 100k, batch 128 |
| `train` | Episodes 3063, episode length 1500 s, master seed 42 |
| `evaluation` | 8 scenarios × 20 seeds, episode length 1800 s, perturbation definitions |
| `visualization` | Output 1600×1600 px, congestion/queue/wait scales, colour map |
